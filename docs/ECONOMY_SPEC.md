## 1. 目的と位置づけ

### 1.1 目的

- バトル勝敗に連動する **経済ループ**（獲得・消費・ロスト・復帰）を定義する。
- 創作（ピクセル絵・カード編集）とバトル継続の **モチベーション設計** を明文化する。
- 無償プレイ・広告・サブスク・単品課金の **役割分担** を固定し、実装・バランス調整の基準とする。

### 1.2 本書とプロトタイプ仕様の関係

| ドキュメント | 役割 |
|--------------|------|
| [PROTOTYPE_DEVELOPMENT_SPEC.md](./PROTOTYPE_DEVELOPMENT_SPEC.md) | 戦闘ルール・UI・技術構成・**仮ロスト**（現行プロトタイプ） |
| **本書（ECONOMY_SPEC.md）** | 通貨・アイテム・本番ロスト・ショップ・課金 |

- 戦闘エンジン・墓地・撃破ルールはプロトタイプ仕様に従う。
- **敗北時の仮ロスト**（§11）は、経済実装完了まで **段階的に併存** し、完了後に本書の Lost 仕様へ切り替える（[§17](#17-プロトタイプからの移行)）。

### 1.3 スコープ

本書は **初版（CPU 戦）から将来機能まで** を章として含む。各機能に **実装段階**（`v1` / `将来`）を付記する。

## 2. 設計原則

1. **デッキは常に最大5枚** — ロスト・削除後は5枚揃うまでバトル不可。
2. **同時未処理ロストは最大1枚** — 1枚ロストでバトル不可のため、未処理ロストが2枚以上になることはない。
3. **バトル報酬だけでは主要アイテムを賄えない** — 不足分は広告視聴・サブスク・単品購入で補う。
4. **創作の努力が経済に反映される** — 塗り数・色数が復活コスト・初期レア抽選に影響する。
5. **有償優先の公平性** — ショップでは無償ピクセル→有償ピクセルの順で消費。一部商品は有償ピクセルのみ購入可。
6. **用語の区別** — 通貨「ピクセル」と、カードの「塗りピクセル数（マス数）」は UI 上で混同しない（[§16](#16-ui--ux-要件)）。

## 3. 通貨

### 3.1 種類

| 種別 | 名称（UI） | 主な獲得源 | 主な用途 |
|------|------------|------------|----------|
| 無償 | **ピクセル**（無償） | バトル勝利、ロスト後削除の返還、広告2倍 | ショップ（対象商品のみ） |
| 有償 | **ピクセル**（有償） | サブスク、将来の通貨パック | ショップ、有償専用商品 |

- 表示上はどちらも **「ピクセル」** と呼ぶ。内訳はアイコン・色・括弧表記等で区別する（[§16](#16-ui--ux-要件)）。
- 有償ピクセルは **返金不可の仮想通貨** として扱う（実装・ストア規約は別途）。

### 3.2 消費順序

ショップ購入時:

1. **無償ピクセル** を先に消費
2. 不足分を **有償ピクセル** から消費

**有償ピクセルのみ** 購入可能な商品がある（[§10.2](#102-商品一覧)）。

### 3.3 上限

- 所持上限は **未定**（実装時に決定。上限なしも選択可）。
- 小数は **切り上げ（ceil）** で整数ピクセルとする（復活ポーション消費量等）。

## 4. バトル報酬・ペナルティ

### 4.1 勝利報酬（`v1`: CPU 戦）

| 項目 | 内容 |
|------|------|
| トリガー | プレイヤー勝利 |
| 操作 | 相手 **墓地** から **プレイヤーが1枚選択** |
| 獲得 | **ゲーム内通貨（ピクセル・無償）のみ** — カードデータは保存しない |
| 墓地が空 | **起こらない**（勝利＝相手が1枚以上撃破済み） |

**通貨量（暫定）**

```
獲得 = BASE_WIN_CURRENCY + RARITY_BONUS[選択カードのレア度]
```

| 定数 | 値 |
|------|-----|
| `BASE_WIN_CURRENCY` | 10 |
| N ボーナス | +10 |
| R ボーナス | +15 |
| SR ボーナス | +20 |

例: 相手墓地の R カードを選ぶ → 10 + 15 = **25 無償ピクセル**。

- 相手墓地のカードレア度は、CPU デッキ生成時のレア（将来は相手カードの `rarity` フィールド）を参照する。
- 勝利後、**広告視聴で無償ピクセル獲得量2倍**（[§11](#11-広告)）。

### 4.2 敗北ペナルティ（`v1` / 将来対人）

| 項目 | 内容 |
|------|------|
| トリガー | プレイヤー敗北 |
| 対象 | 自軍 **墓地** から1枚 |
| CPU 戦 | **ランダム** に1枚選定 |
| 対人戦 | **相手ユーザー** が墓地から1枚選択（[§13](#13-対人戦将来)） |
| 自軍墓地が空 | **起こらない**（敗北＝自軍が1枚以上撃破済み） |
| 結果 | 選ばれた1枚が **Lost** 状態へ（護符があれば [§7](#7-護符)） |

### 4.3 戦績

- 勝利 / 敗北時、出撃した5枚の `wins` / `losses` を従来どおり更新する（プロトタイプ仕様 §11.3 準拠）。

### 4.4 EXP

- EXP 加算ルールは **プロトタイプ仕様のまま**（経済実装と独立）。

## 5. 墓地と戦闘終了処理

### 5.1 墓地のスコープ

- **その戦闘中に撃破され墓地へ送られたカードのみ** が対象。
- 戦闘をまたいだ墓地の持ち越しは **ない**。

### 5.2 戦闘終了後のカード状態

| カード | 戦後状態 |
|--------|----------|
| ロスト対象に選ばれた1枚 | `status: lost` |
| 護符で免れた1枚 | `status: active`（護符消滅） |
| その他の自軍5枚 | `status: active`（通常どおりデッキで使用可） |

- 勝利時: 自軍5枚はすべて `active`。相手墓地から通貨化した1枚は相手側データのみ参照（自デッキ変更なし）。

### 5.3 演出要件

敗北時、**どのカードがロスト対象になったか** を明示する。

| 対戦 | 演出 |
|------|------|
| CPU | 墓地から選ぶ **スロット／ルーレット風** 演出 |
| 対人 | 相手が墓地から選んだカードが **相手操作として可視** |

## 6. Lost（ロスト）状態と復帰

### 6.1 Lost の定義

- カードデータは **削除しない**（Lost プール相当）。
- `status: lost` のカードは **バトル・編集不可**（編集可否は [§20](#20-未決定要調整)）。
- アクティブが5枚未満の間は **バトル開始不可** — これ自体がペナルティの一部。

### 6.2 処理 UI

| 項目 | 内容 |
|------|------|
| タイミング | 敗北直後は **次バトル不可**。 **マイデッキ画面** で処理 |
| 表示 | ロストカードに **ロスト状態の見た目** ＋ 選択ボタン |

### 6.3 復帰オプション

ロストカードごとに、レア度に応じて以下から **1つ** を選ぶ。

#### 6.3.1 完全復活（全レア）

| 項目 | 内容 |
|------|------|
| 手段 | **復活ポーション** を消費 |
| `reviveCount` | **+1**（カードに表示） |
| レア度 | **維持** |
| HP / ★ | **維持** |

**ポーション消費量**

```
消費 = ceil((塗り数 + HP) × FULL_REVIVE_MULTIPLIER[rarity])
```

| レア | 倍率 |
|------|------|
| N | 1.0 |
| R | 1.2 |
| SR | 1.5 |

- `塗り数` = 16×16 グリッドで `#RRGGBB` が入っているマス数（最大256）。

#### 6.3.2 降格復活（R / SR のみ）

| 項目 | 内容 |
|------|------|
| 手段 | **復活ポーション** を消費（完全復活より少ない量） |
| `reviveCount` | **増やさない** |
| レア度 | **1段階降格**（SR→R、R→N） |
| HP / ★ | **降格に応じて減少**（係数表は [§9.4](#94-hp-と表示)） |

```
消費 = ceil((塗り数 + HP) × DOWNGRADE_REVIVE_MULTIPLIER[rarity])
```

| レア（ロスト時） | 倍率 |
|------------------|------|
| R | 1.0 |
| SR | 1.2 |

#### 6.3.3 削除（全レア）

| 項目 | 内容 |
|------|------|
| 手段 | 削除ボタン（**溶解リキッド不要** — ロスト専用経路） |
| データ | カード **完全削除** |
| 報酬 | 無償ピクセル = `ceil(塗り数 × 0.10)`（例: 256塗り → 26） |
| その後 | **新規カード作成が必須**（5枚揃えるまでバトル不可） |

#### 6.3.4 N ロスト時の選択肢

- **完全復活** または **削除** の **2択のみ**（降格復活なし）。

## 7. 護符

**実装段階: `v1`**

### 7.1 概要

- カードに装備可能な消費アイテム。
- そのカードが **ロスト対象に選ばれたとき**、自動発動し **1回だけロストを免れる**。

### 7.2 ルール

| 項目 | 内容 |
|------|------|
| 装備数 | **複数枚同時装備可**（カードごとに0〜1個想定） |
| 発動 | ロスト対象選定時 **自動** → 護符消滅、カードは **active のまま** |
| 対人戦 | **有効** |
| 復帰 UI | 免れた場合、§6.3 の選択は **不要** |

### 7.3 取得

- ショップ購入（[§10](#10-アイテムとショップ)）
- サブスク定額配布（[§12](#12-課金サブスクリプション)）

## 8. カード削除

### 8.1 通常削除（ロスト以外）

**実装段階: `v1`**

| 項目 | 内容 |
|------|------|
| 手段 | **溶解リキッド** 1個消費 |
| 場所 | マイデッキ **編集モード** および **デッキ管理画面**（統一） |
| 報酬 | **なし** |
| 削除後 | **新規カード作成必須**（5枚維持） |

- プロトタイプの「編集モードで無条件削除」は **本仕様実装後に廃止** する。

### 8.2 削除経路の整理

| 経路 | 条件 | コスト | 報酬 |
|------|------|--------|------|
| ロスト後削除 | `status: lost` | なし | 無償ピクセル（塗り10%） |
| 通常削除 | `active` | 溶解リキッド | なし |

## 9. レア度・初期抽選・限界突破

### 9.1 レア段階

| 段階 | 記号 | 実装段階 |
|------|------|----------|
| Normal | N | `v1` |
| Rare | R | `v1` |
| Super Rare | SR | `v1` |
| Ultra Rare | UR | 将来 |
| Legend | L | 将来 |

### 9.2 新規作成時のレア抽選

**実装段階: `v1`（経済フェーズ2以降推奨）**

#### 9.2.1 デフォルト確率

| レア | 確率 |
|------|------|
| N | 80% |
| R | 15% |
| SR | 5% |

#### 9.2.2 創作ボーナス（条件達成時）

**条件（すべて満たす）**

1. 塗りピクセル比率 **≥ 80%**（`塗り数 / 256 ≥ 0.8`）
2. 使用色数 **＝** 当該ユーザーレベルで **解放済みの色数**

条件達成時の抽選:

| レア | 確率 |
|------|------|
| N | 50% |
| R | 35% |
| SR | 15% |

- ユーザーレベルとパレット解放の対応はプロトタイプ仕様 §5 / `PALETTE_16` に従う。
- **追加の細かい条件**（HP 帯・対称性等）は将来拡張可（[§20](#20-未決定要調整)）。

#### 9.2.3 既存セーブへの遡及

- **未定** — 経済実装時に決定。推奨案: 既存カードは **N・★0** として移行し、新規作成のみ抽選。

### 9.3 限界突破（★）

**実装段階: `将来`（仕様確定済み）**

| 項目 | 内容 |
|------|------|
| 手段 | **限界突破アイテム**（専用） |
| 同一レア内 | ★1 → ★2 → ★3（最大 **3回**） |
| 4回目相当 | **レア1段階アップ** ＋ HP 増 ＋ ★リセット（★0） |
| 初版上限 | **UR 実装まで SR★3 が上限**（4回目昇格は不可） |
| 表示 | カードに **★1〜3** を表示 |

**昇格例（UR 実装後）**

```
N★3 --(4回目)--> R★0
R★3 --(4回目)--> SR★0
SR★3 --(4回目)--> UR★0  （UR 実装後）
...
```

### 9.4 HP と表示

| イベント | HP |
|----------|-----|
| ★1〜3 付与 | **微増**（係数未定） |
| レア昇格 | **大増**（係数未定） |
| 降格復活 | 降格後レアの基礎 HP へ再計算 |

- 基礎 HP は現行どおり `card/` モジュール（色バランス + hash）で算出し、`rarity` / `stars` 倍率を乗算する方式を推奨。
- レア枠・ラベル・★表示は **オーバーレイ**（ピクセル本体は原則変更しない）。

## 14. 将来拡張（仕様予約）

### 14.1 アルバム

- 勝利時に通貨化した相手カードの **メタデータ・サムネ** をコレクションとして保存。
- デッキ5枚制限とは独立。

### 14.2 補欠枠

- デッキ外の **第2デッキ / ストック枠**。
- 現行「ストックなし」と矛盾するため **課金または高ランク報酬** 想定。

### 14.3 UR / Legend

- レア段階追加に伴い、勝利ボーナス表・復活倍率表・限界突破上限を拡張。

## 15. データモデル

### 15.1 Card（拡張案）

```typescript
type CardRarity = "N" | "R" | "SR" | "UR" | "L";
type CardStatus = "active" | "lost";

interface Card {
  id: string;

  pixels: PixelGrid;
  attribute: Attribute;
  hp: number;
  baseHp?: number;           // 限界突破・降格用（将来）

  stars: 0 | 1 | 2 | 3;      // 限界突破（将来）
  status: CardStatus;          // active | lost
  talismanEquipped: boolean;   // 護符装備
}
```

```typescript
interface UserEconomy {
```

## 16. UI / UX 要件

### 16.1 通貨表示

- ヘッダー等に **無償 / 有償** の内訳または合計＋内訳ツールチップ。

- サブスクプラン比較表（§12.2）。

- 仮ロスト併存期間は「プロトタイプ：カードは保存されています」注記を **本仕様 UI 実装完了まで** 維持可。
