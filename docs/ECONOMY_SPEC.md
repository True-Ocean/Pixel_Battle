## 1. 目的と位置づけ

### 1.1 目的

- バトル勝敗に連動する **経済ループ**（獲得・消費・ロスト・復帰）を定義する。
- 創作（ピクセル絵・カード編集）とバトル継続の **モチベーション設計** を明文化する。
- 無償プレイ・広告・サブスク・単品課金の **役割分担** を固定し、実装・バランス調整の基準とする。

### 1.2 本書とプロトタイプ仕様の関係

| ドキュメント | 役割 |
|--------------|------|
| [PROTOTYPE_DEVELOPMENT_SPEC.md](./PROTOTYPE_DEVELOPMENT_SPEC.md) | 戦闘ルール・UI・技術構成・**仮ロスト**（現行プロトタイプ） |
| **本書（ECONOMY_SPEC.md）** | 通貨・アイテム・本番ロスト・ショップ・課金 |
| [ECONOMY_ROADMAP.md](./ECONOMY_ROADMAP.md) | 段階的実装順序・フェーズ定義 |

- 戦闘エンジン・墓地・撃破ルールはプロトタイプ仕様に従う。
- **敗北時の仮ロスト**（§11）は、経済実装完了まで **段階的に併存** し、完了後に本書の Lost 仕様へ切り替える（[§17](#17-プロトタイプからの移行)）。

### 1.3 スコープ

本書は **初版（CPU 戦）から将来機能まで** を章として含む。各機能に **実装段階**（`v1` / `将来`）を付記する。

## 2. 設計原則

1. **デッキは常に最大5枚** — ロスト・削除後は5枚揃うまでバトル不可。
2. **同時未処理ロストは最大1枚** — 1枚ロストでバトル不可のため、未処理ロストが2枚以上になることはない。
3. **バトル報酬だけでは主要アイテムを賄えない** — 不足分は広告視聴・サブスク・ジュエル購入で補う。
4. **創作の努力が経済に反映される** — 塗り数・色数が復活コスト・初期レア抽選・戦利品 px に影響する。
5. **操作地点での直消費** — 削除・リネーム・デッキ3以降は **💎 ジュエルをその場で消費**。中間 consumable（溶解リキッド等）は使わない。
6. **限界突破は専用資源** — px・💎 では代替不可。**属性かけら**（バトル戦利品）と **汎用かけら**（Lv20 以降の節目報酬）の二系統。
7. **用語の区別** — 通貨「ピクセル」と、カードの「塗りピクセル数（マス数）」は UI 上で混同しない（[§16](#16-ui--ux-要件)）。
8. **序盤の学習期間** — ユーザーレベル **5 未満** は敗北ロストを発生させない（[§4.2.1](#421-初心者保護ユーザーレベル--5)）。**Lv5 到達** と同時に Lost を解禁し、初回 **護符 ×1** を配布する（[§7.3](#73-取得)、[§10.4](#104-レベルアップ報酬)）。

## 3. 通貨

### 3.1 種類

| 種別 | 名称（UI） | 主な獲得源 | 主な用途 |
|------|------------|------------|----------|
| 無償 | **ピクセル**（px） | バトル勝利、レベルアップ、**カード削除返還**、勝利2倍広告 | **復活**、護符購入（px 枠）、**リネーム初回**、編集時キャンバス拡大 |
| 有償 | **💎 ジュエル** | 課金、**毎レベル少量**、L≡4 (mod 5) ボーナス、サブスク | **カード削除**・リネーム（2回目以降）・**デッキ3以降**・限界突破レア昇格・創作拡張の **その場消費** |
| 専用 | **属性かけら** | 勝利時の戦利品選択（§4.1） | 同一属性カードの限界突破（**N=10 / R=15 / SR・UR=20**） |
| 汎用 | **汎用かけら** | Lv20, 30, 40… のレベルアップ（§10.4） | 任意属性のかけらとして消費（属性かけらと**同価値**） |
| 消費品 | **護符** | **Lv5 到達（初回 ×1）**、ショップ、サブスク | ロスト1回免れ（装備消費） |

- px と 💎 はヘッダー等で **アイコンを分けて** 常時表示する。
- 💎 が必要な操作にはボタンに **💎 と数値** を併記する（中間チケット名は使わない）。
- 属性かけらは **所持品タブ** に一覧表示。マイデッキ詳細の限界突破は [§16.9](#169-限界突破ui) 参照。
- ジュエルは **返金不可の仮想通貨** として扱う（実装・ストア規約は別途）。

### 3.2 廃止したモデル

以下は **採用しない**（2026-06 設計改定）:

- 有償ピクセル / 無償→有償の消費順
- 復活ポーション・溶解リキッド・リネームチケット・デッキ解放キー等の **中間 consumable 商品**

### 3.3 上限

- px・💎 の所持上限は **未定**（実装時に決定。上限なしも選択可）。
- 復活等の px 消費は **整数**（`Math.floor`）。カードごとに塗り・レア・★で変動（§18.1）。

## 4. バトル報酬・ペナルティ

### 4.1 勝利報酬（`v1`: CPU 戦）

| 項目 | 内容 |
|------|------|
| トリガー | プレイヤー勝利 |
| 操作 | 相手 **墓地** から **プレイヤーが1枚選択** |
| 獲得 | **無償ピクセル** ＋ **選択カード属性のかけら**（レア度別・[§9.3](#93-限界突破)） |
| カードデータ | 自デッキには **保存しない**（相手側参照のみ） |
| 墓地が空 | **起こらない**（勝利＝相手が1枚以上撃破済み） |

**生存報酬（確定）**

- 生存枚数 × `PIXELS_PER_SURVIVOR`（プロトタイプ: **10 px/枚**）。

**戦利品 px（プロトタイプ実装）**

```
px = floor(√塗り数 × K × 色係数)
```

- 実装: `calcGraveyardPixelReward`（`src/config/economy.ts`）。

**戦利品 px（将来・レアボーナス案）**

```
獲得 = BASE_WIN_CURRENCY + RARITY_BONUS[選択カードのレア度]
```

| 定数 | 値 |
|------|-----|
| `BASE_WIN_CURRENCY` | 10 |
| N ボーナス | +10 |
| R ボーナス | +15 |
| SR ボーナス | +20 |

**戦利品・属性かけら**

| 項目 | 内容 |
|------|------|
| 基本 | 選択カードの `attribute` に対応するかけら（N=1, R=2, SR=3） |
| レアボーナス | 上記で確定（`GRAVEYARD_SHARD_REWARD`） |
| 限界突破消費 | **かけら合計**（専用＋汎用、1:1。px・💎不可）。**N=10 / R=15 / SR・UR=20** |

- 相手墓地のレア度は **バトル中の `BattleUnit.rarity`** を正とし、戦利品モーダルへ引き継ぐ（`resolveGraveyardLootCards`）。
- バトル準備・対戦中は R/SR+ カードを **枠色＋背景色** で表示（`BattleCard`）。
- マイデッキ詳細では、**かけらが10個以上揃い突破可能なときだけ**、編集・削除ボタンの直上に限界突破 UI を表示（不足時は非表示）。
- 勝利後、**任意**でリワード広告 → 当該戦の **px のみ ×2**（かけらは2倍にしない）（[§11.1](#111-勝利後2倍)）。
- 戦利品選択 UI: `GraveyardPickModal` — px とかけらをカードごとに表示。

### 4.1.1 バトル回数制限（非会員）

| 項目 | 内容 |
|------|------|
| 対象 | 会員未加入の無課金ユーザー |
| 無料枠 | **1日10戦** まで CPU 戦開始可 |
| 超過時 | **リワード広告視聴** で1戦追加（繰り返し可） |
| リセット | 日次（タイムゾーン TBD） |
| 会員 | ライト: 軽減 / プレミアム: 上限なしまたは広告非表示（[§12.2](#122-配布内容)） |
| 対象外 | **バトル履歴再戦**（[§4.1.2](#412-バトル履歴再戦v1-プロトタイプ)）は日次 cap に **カウントしない** |

### 4.1.2 バトル履歴再戦（`v1` プロトタイプ）

**実装段階: `v1`（プロトタイプ）**

| 項目 | 内容 |
|------|------|
| 入口 | **戦績** → **バトル履歴** → 対戦詳細 →「もう一度対戦する」 |
| 初回告知 | **再戦時のルール** モーダル（`HistoryRematchRulesModal`）。「今日はこれ以降表示しない」→ JST 日次キー `historyRematchRulesDismissedDayKey` |
| 広告 | **3回に1回**、ルールモーダル後・**デッキ選択の前** にモックリワード広告（[§11.4](#114-履歴再戦モック)） |
| デッキ選択 | 通常 CPU 戦と同一 UI（`BattleDeckSelectScreen`）。**5枚揃い** なら **ロストカード含め** 出撃可（`deckReadinessMode="historyRematch"`） |
| 相手デッキ | 履歴の **カード構成は維持**。**BP のみ** 現在のユーザーレベルで再計算（`prepareHistoryOpponentDeck`） |
| マッチング演出 | **スキップ**（相手確定済み） |
| 履歴追記 | **しない** |
| 戦績 | カード `wins` / `losses`・ユーザー戦績は **更新しない** |
| EXP | **加算しない** |
| 敗北 | **ロスト抽選なし**（`LostRoulette` スキップ） |
| 勝利報酬 | **生存カード分の px のみ**（墓地選択・属性かけら・勝利2倍広告は対象外）。バトル終了 UI の中央ガイダンス帯に **+N** をインライン表示 |
| 勝敗後 |「もう一度対戦」→ 再びデッキ選択フロー（ルールモーダル・広告判定を含む） |

- UI フロー詳細は [PROTOTYPE §4.6](./PROTOTYPE_DEVELOPMENT_SPEC.md#46-戦績バトル履歴)・[§11.4](./PROTOTYPE_DEVELOPMENT_SPEC.md#114-バトル履歴再戦) 参照。

### 4.2 敗北ペナルティ（`v1` / 将来対人）

| 項目 | 内容 |
|------|------|
| トリガー | プレイヤー敗北 |
| 対象 | 自軍 **墓地** から1枚 |
| CPU 戦 | **ランダム** に1枚選定 |
| 対人戦 | **相手ユーザー** が墓地から1枚選択（[§13](#13-対人戦将来)） |
| 自軍墓地が空 | **起こらない**（敗北＝自軍が1枚以上撃破済み） |
| 結果 | 選ばれた1枚が **Lost** 状態へ（護符があれば [§7](#7-護符)） |
| 前提 | **ユーザーレベル ≥ 5**（未満は [§4.2.1](#421-初心者保護ユーザーレベル--5)） |

### 4.2.1 初心者保護（ユーザーレベル &lt; 5）

**実装段階: `v1`**

| 項目 | 内容 |
|------|------|
| 条件 | **ユーザーレベル &lt; 5** の間 |
| 敗北時 | **ロスト抽選を行わない** — 自軍墓地からの1枚選定・`LostRoulette` 演出・`status: lost` 化をすべてスキップ |
| 戦績 | 敗北として `losses` は加算してよい（[PROTOTYPE §11.3](./PROTOTYPE_DEVELOPMENT_SPEC.md#113-戦績) 準拠） |
| 対象 | **CPU 戦・練習戦** を問わない |
| 解禁 | **Lv5 到達時** から §4.2 の通常ルールを適用。同時に [§7.3](#73-取得) の初回護符を配布し、レベルアップ UI で Lost 解禁を告知する（[§16.8](#168-レベル5到達時lost-解禁)） |
| 既存セーブ | Lv5 以上のユーザーには **遡及しない**（新規および Lv5 未満のプレイのみ対象） |

### 4.3 戦績

- 勝利 / 敗北時、出撃した5枚の `wins` / `losses` を従来どおり更新する（プロトタイプ仕様 §11.3 準拠）。
- **バトル履歴再戦**（[§4.1.2](#412-バトル履歴再戦v1-プロトタイプ)）では **更新しない**。

### 4.4 EXP

- EXP 加算ルールは **プロトタイプ仕様のまま**（経済実装と独立）。
- **バトル履歴再戦**では EXP を **加算しない**（[§4.1.2](#412-バトル履歴再戦v1-プロトタイプ)）。

## 5. 墓地と戦闘終了処理

### 5.1 墓地のスコープ

- **その戦闘中に撃破され墓地へ送られたカードのみ** が対象。
- 戦闘をまたいだ墓地の持ち越しは **ない**。

### 5.2 戦闘終了後のカード状態

| カード | 戦後状態 |
|--------|----------|
| ロスト対象に選ばれた1枚 | `status: lost` |
| 護符で免れた1枚 | `status: active`（護符消滅） |
| その他の自軍5枚 | `status: active`（通常どおりデッキで使用可） |

- 勝利時: 自軍5枚はすべて `active`。相手墓地から通貨化した1枚は相手側データのみ参照（自デッキ変更なし）。
- **ユーザーレベル &lt; 5** で敗北した場合: 自軍5枚はすべて `active` のまま（§4.2.1）。

### 5.3 演出要件

敗北時、**どのカードがロスト対象になったか** を明示する。

| 対戦 | 演出 |
|------|------|
| CPU | 墓地から選ぶ **スロット／ルーレット風** 演出 |
| 対人 | 相手が墓地から選んだカードが **相手操作として可視** |

## 6. Lost（ロスト）状態と復帰

### 6.1 Lost の定義

- カードデータは **削除しない**（Lost プール相当）。
- `status: lost` のカードは **バトル・編集不可**（編集可否は [§20](#20-未決定要調整)）。
- アクティブが5枚未満の間は **バトル開始不可** — これ自体がペナルティの一部。

### 6.2 処理 UI

| 項目 | 内容 |
|------|------|
| タイミング | 敗北直後は **次バトル不可**。 **マイデッキ画面** で処理 |
| 表示 | ロストカードに **ロスト状態の見た目** ＋ 選択ボタン |

### 6.3 復帰オプション

ロストカードごとに、以下から **1つ** を選ぶ（復活上限未到達時）。**降格復活は採用しない**（2026-06-20 廃止）。

#### 6.3.1 復活（全レア）

> **プロトタイプ**: 無償 px を **カードごと** に消費（塗り×3×レア×★）。詳細は [§18.1](#181-プロトタイプ簡略実装現行コード)。

| 項目 | 内容 |
|------|------|
| 手段 | **無償ピクセル** をその場で消費 |
| `reviveCount` | **+1**（カードに **「復活 n/3」** と表示） |
| 復活上限 | **`REVIVE_CAP = 3`**（`reviveCount >= 3` では px 復活不可） |
| 護符 | ロスト免れ時は **`reviveCount` を増やさない** |
| レア度 | **維持** |
| BP / ★ | **維持** |
| UI 表記 | ボタン・確認は **「復活」**（「完全復活」は使わない） |

~~**ポーション消費量**~~（**廃止** — 2026-06 設計改定）

#### 6.3.2 降格復活（R / SR のみ）

> **廃止**（2026-06-20）。プロトタイプでは **復活・思い出アルバムに保存・削除** のみ。

~~**プロトタイプ**: 無償 px を **カードごと** に消費。傾斜は **復活後レア** と **現★** を基準（§18.1）。★は **維持**、BP は降格後レアで再算出。~~

#### 6.3.3 削除（全レア・active / lost 共通）

> **プロトタイプ**: **💎 1** 消費 ＋ **px・属性かけら返還**（返還 px は勝利戦利品と同式）。詳細は [§18.1](#181-プロトタイプ簡略実装現行コード)。

| 項目 | 内容 |
|------|------|
| 手段 | 削除ボタン（**💎 消費**） |
| データ | カード **完全削除**（デッキから除去） |
| 報酬 | **px**（`calcGraveyardPixelReward`）＋ **属性かけら**（N=1, R=2, SR=3） |
| 確認 | **二段階**（返還内容を 💎・px・かけらアイコン付きで表示） |
| その後 | **新規カード作成が必須**（5枚揃えるまでバトル不可） |

#### 6.3.4 思い出アルバムに保存（全レア・active / lost 共通）

> **プロトタイプ**: **無償**。デッキから外し **閲覧専用アーカイブ** へ移す。詳細は [§14.1](#141-思い出アルバムプロトタイプ実装済)。

| 項目 | 内容 |
|------|------|
| 手段 | **思い出アルバムに保存** ボタン |
| コスト | **なし**（px・💎 消費なし。**返還もなし**） |
| データ | デッキから **除去** → `memoryAlbum.cards` に追加（`status: active` で保存） |
| 確認 | **1段階**（「デッキに戻せず閲覧のみ」旨を表示） |
| 復帰 | **デッキへは戻せない**（アルバムから **💎1 削除** で px・かけら返還のみ） |

#### 6.3.5 ロスト時の選択肢（プロトタイプ UI）

| 状況 | 選択肢 |
|------|--------|
| 復活上限 **未到達** | **復活** / **思い出アルバムに保存** / **削除** |
| 復活上限 **到達** | **思い出アルバムに保存** / **削除**（「復活上限に達しました」表示） |
| N ロスト | 上記と同様（降格復活なし） |

#### 6.3.6 active カードのデッキから外す操作

| 操作 | 挙動 |
|------|------|

## 7. 護符

**実装段階: `v1`**

### 7.1 概要

- カードに装備可能な消費アイテム。
- そのカードが **ロスト対象に選ばれたとき**、自動発動し **1回だけロストを免れる**。

### 7.2 ルール

| 項目 | 内容 |
|------|------|
| 装備数 | **複数枚同時装備可**（カードごとに0〜1個想定） |
| 発動 | ロスト対象選定時 **自動** → 護符消滅、カードは **active のまま** |
| 対人戦 | **有効** |
| 復帰 UI | 免れた場合、§6.3 の選択は **不要** |

### 7.3 取得

| 経路 | 内容 |
|------|------|
| **Lv5 到達（初回のみ）** | **護符 ×1** を無償配布（`inventory.talisman += 1`）。生涯1回。Lost 解禁（§4.2.1）と **同時** に付与し、レベルアップ UI で説明する |

| サブスク | 定額配布（[§12](#12-課金サブスクリプション)） |

- Lv5 以前はロストが発生しないため、初回護符は **所持のみ**（装備・消費は Lv5 以降の敗北から）。

### 9.2 新規作成時のレア抽選

**実装段階: `v1`（経済フェーズ2以降推奨）**

#### 9.2.1 デフォルト確率

| レア | 確率 |
|------|------|
| N | 80% |
| R | 15% |
| SR | 5% |

#### 9.2.2 創作ボーナス（条件達成時）

**条件（すべて満たす）**

1. 塗りピクセル比率 **≥ 80%**（`塗り数 / (canvasSize²) ≥ 0.8`）
2. 使用色数 **＝** 当該ユーザーレベルで **解放済みの色数**

条件達成時の抽選:

| レア | 確率 |
|------|------|
| N | 50% |
| R | 35% |
| SR | 15% |

- ユーザーレベルとパレット解放の対応はプロトタイプ仕様 §5.6 に従う（Lv50 上限で **レベル解放8色** ＋ **ショップ12色** で計20色）。
- キャンバスサイズは §5.7 のマイルストーンに従い、カードの `canvasSize` で密度を算出する。
- **属性決定への追加色反映**・創作ボーナスの細かい調整は将来拡張可（[§20](#20-未決定要調整)）。

#### 9.2.3 既存セーブへの遡及

- **未定** — 経済実装時に決定。推奨案: 既存カードは **N・★0** として移行し、新規作成のみ抽選。

### 9.3 限界突破（★）

**実装段階: `v1`（プロトタイプ）**

| 項目 | 内容 |
|------|------|
| 手段 | **かけら合計**（専用＋汎用、**1個=1価値**）。★アップ時はかけらのみ。**N=10 / R=15 / SR・UR=20** |
| レア昇格（4回目相当） | 上記かけらに加え **💎 追加**（`LIMIT_BREAK_RARITY_JEWEL_COST`: N→R **10** / R→SR **20** / SR→UR相当 **40**） |
| UI | 詳細オーバーレイで **専用・汎用の使用数を個別選択**（合計は現在レア度に応じる）。表示は [§16.9](#169-限界突破ui) |
| 同一レア内 | ★1 → ★2 → ★3（最大 **3回**） |
| 4回目相当 | **レア1段階アップ** ＋ BP 増 ＋ ★リセット（★0） |
| 初版上限 | **UR 実装まで SR★3 が上限**（4回目昇格は不可） |
| 表示 | カードに **★1〜3** を表示 |

**資源の種類**

| 種類 | 入手 | 使用対象 |
|------|------|----------|
| **属性かけら** | 勝利時の **戦利品選択**（§4.1）。選択カードの属性に +1 | **同一属性**のカードのみ |
| **汎用かけら** | レベル **L ≡ 0 (mod 10)** かつ **L ≥ 20**（20, 30, 40, 50 …） | **全属性**のカード（属性かけらの代用） |

- **Lv10 では汎用かけらを配布しない**（デッキ2解放に枠を使用。§10.4）。
- 限界突破1回につき **現在レア度に応じたかけら数**（`getLimitBreakShardsRequired`）。専用・汎用は **1:1** で合算。
- 実行時はプレイヤーが選択した内訳で消費（ステッパーで調整。合計は常に10）。
- 戦利品かけらは **N=1, R=2, SR=3**（`GRAVEYARD_SHARD_REWARD`）。
- 将来ショップ: 属性かけらパック（💎）は **ショートカット** として検討可。

**戦略的意図**

- 育成したい属性のカードがある場合、相手墓地から **同属性の戦利品** を選ぶ動機が生まれる。
- px 最大化と属性かけらの **トレードオフ** が戦利品 UI に集約される。

**昇格例（UR 実装後）**

```
N★3 --(4回目)--> R★0
R★3 --(4回目)--> SR★0
SR★3 --(4回目)--> UR★0  （UR 実装後）
...
```

### 9.4 BP と表示

| イベント | BP |
|----------|-----|
| ★アップ・レア昇格 | **均等加算**（`LIMIT_BREAK_BP_GAIN_RATE` × 基礎BP、1回ごとに同じ増加量） |
| ~~降格復活~~ | ~~降格後レアの基礎 BP へ再計算~~（**廃止** — 2026-06-20） |

- 基礎 BP は `getCardFoundationBp`（色バランス + hash、レア・★適用前）。限界突破のたびに `round(基礎BP × 0.03)` を加算（最低1）。
- N★0→SR★3 は **11回** の限界突破。基礎BP=100 なら **1回 +3 BP** で最終 **133**。
- レア枠・ラベル・★表示は **オーバーレイ**（ピクセル本体は原則変更しない）。

- 新規作成時のサイズ変更は無料（縮小・拡大とも選択可）
- 実装: `EditorScreen.tsx` / `CanvasSizePicker.tsx`

### 18.1 プロトタイプ簡略実装（現行コード）

CPU 戦プロトタイプでは **`src/config/economy.ts`** の式を使用する。**復活ポーションは採用しない**（§6.3）。

#### 復活 px（`calcFullReviveCost`）

```
floor( 塗りマス数 × REVIVE_PAINTED_MULTIPLIER × レア倍率 × ★倍率 )
```

| 定数 / 倍率 | 値 |
|-------------|-----|
| `REVIVE_PAINTED_MULTIPLIER` | **3**（例: 塗り256 → 基礎768） |
| `REVIVE_CAP` | **3**（`reviveCount` が上限に達すると px 復活不可） |
| レア倍率 | `LOST_WEIGHT_RARITY` — N=1.0, R=1.3, SR=1.6（UR/L は SR 扱い） |
| ★倍率 | `LOST_WEIGHT_STARS` — ★0=1.0, ★1=1.15, ★2=1.3, ★3=1.5 |
| 塗り0 | 最低 **1px** |
| 丸め | **切り捨て**（`Math.floor`） |

| 操作 | 傾斜の基準 |
|------|------------|
| **復活** | **現レア**・**現★** |
