## 1. 目的とスコープ

### 1.1 目的

- 全属性（剣・盾を含む **11 属性**）の **戦闘中の行動・効果・数値** を単一の参照源として定義する。
- [プロトタイプ開発指示書 §8](./PROTOTYPE_DEVELOPMENT_SPEC.md#8-戦闘ルール) の骨格（5枚・前衛2/後衛3・1ターン）に載る **属性レイヤ** を記述する。

### 1.2 プロトタイプ現行との関係

| 区分 | 内容 |
|------|------|
| **実装済み** | 剣・盾・力・弓・両・毒・癒・氷・嵐・忍・照（`attack`〜`illuminate`） |
| **仕様草案・未実装** | 蘇・鏡・爆・重・呪・替・消・増・鎧・先・解 等（[§11](#11-将来属性草案)） |
| **属性決定（作成時）** | **解放済み属性から权重付きランダム**（直近解放属性 +10% ブースト）。色・hash は **不使用**（[§3.2](#32-作成時の属性)） |
| **属性変更（作成後）** | カード詳細の **属性リタッチ**（px）・**属性セレクト**（💎）。[ECONOMY_SPEC §8.4](./ECONOMY_SPEC.md#84-属性変更属性リタッチ属性セレクト) |
| **表示** | 漢字1文字 + `ATTRIBUTE_META`（`src/config/attributes.ts`）。カード詳細の **簡易説明**・**詳細説明（battleGuide）** と、用語タップ時の **コンパクトモーダル**（`src/config/battleGuideCommon.ts` — §2.5） |

### 1.3 前衛・後衛の基本

- **前衛**: 原則 **近接攻撃** が可能（属性ごとに追加行動あり。§4 参照）。
- **後衛**: 属性ごとに行動可否が異なる（§4 参照）。
- **将来方針**: 前衛でも「近接 OR 特殊」の **1ターン1行動** に統一する案あり（現仕様は **属性ごと** に前衛/後衛の行動を定義）。

## 2. 共通ルール

### 2.1 1ターン1行動（行動カード）

| 項目 | 内容 |
|------|------|
| 原則 | 各陣営 **1ターンに1枚だけ** 行動カードを選ぶ |
| 行動の種類 | 近接攻撃・弓攻撃・両・嵐・癒・盾付与 など（属性による） |
| 排他 | 嵐・弓・両・近接は **そのターンの行動カード1枚の攻撃行動**。他カードは同ターン行動しない |
| 付与系 | 毒・氷は **近接の行動カード** で戦った相手に、攻撃と同時に付与（毒は一方的な反撃でも付与 — §4.6） |
| 忍（潜伏状態中） | 後衛は **行動カードに選べない**。前衛は近接可（攻撃で解除） |

### 2.2 ダメージの基準

| 種別 | 基準 |
|------|------|
| **原則** | バトル中の攻撃・付与ダメージ（近接、弓、両副攻撃、嵐など）は **付与者の currentBp** |
| **例外（毒 DoT）** | 各毒スタックは **付与時点の付与元 currentBp × 30%**（§5.1） |
| **回復上限** | 癒による回復は **maxBp まで**（オーバーヒールなし） |
| **表示** | 対人戦で相手が読みやすいよう、攻撃は原則 **今カードに表示されている BP** を基準とする |

### 2.3 近接攻撃と反撃

| 状況 | 攻撃側 | 反撃側 |
|------|--------|--------|
| **相打ち**（同じ2枚が互いに近接の行動カード） | currentBp **100%** | currentBp **100%** |
| **一方的**（片方だけが近接の行動カード、または別の相手を攻撃中の前衛が横から殴られる） | currentBp **100%** | currentBp **50%** |

- 相打ち判定: 同一ターンに互いが相手を近接ターゲットに選んだ **同じ2枚** の組（実装: `bidirectional` フラグ）。
- 弓・両副攻撃・嵐は **反撃ルールの対象外**（無反撃または別規定）。

### 2.4 盾（全属性共通）

| 項目 | 内容 |
|------|------|
| 効果 | **1回限り**、ダメージを 0 にして消費 |
| 嵐 | 盾があれば **盾消費・ダメージ 0** |
| 攻撃側の盾 | 近接で攻撃した側が盾を持つ場合、反撃ダメージ 0・攻撃側盾消費（剣/盾 Lv0 ルール維持）。**凍結中の相手**への近接は反撃がないため **盾は消費しない** |

### 2.5 カード詳細の戦い方表示（UI）

カード詳細（マイデッキ・編集画面）では、戦闘ルールの **要約** を `ATTRIBUTE_META` で表示し、詳細は **用語タップ** でモーダル表示する（`GuideTextWithTerms` / `BattleTermGuideModal`）。

| 項目 | ソース | 内容 |
|------|--------|------|
| **簡易説明** | `ATTRIBUTE_META.description` | 属性枠下の1〜2行テキスト |
| **詳細説明** | `ATTRIBUTE_META.battleGuide` | 折りたたみ「詳細説明」本文（`white-space: pre-line`） |
| **用語リンク** | `LINKABLE_BATTLE_GUIDE_TERMS` | 詳細説明内の用語をタップ可能に（長いラベル優先でマッチ） |
| **モーダル** | `BATTLE_GUIDE_TERMS` | タップ時に `BattleTermGuideModal` を表示。`compact: true` はコンパクトパネル |

**近接攻撃**（`melee`）のみフルサイズモーダル。それ以外の用語はコンパクトモーダル（箇条書き・小さめパネル）。

#### 2.5.1 各属性の UI 文案（`ATTRIBUTE_META`）

| 属性 | 簡易説明 | 詳細説明（battleGuide） |
|------|----------|-------------------------|
| 剣 | 通常攻撃のみ（特殊能力なし） | ・前衛にいるときのみ近接攻撃できる（後衛は行動不可）<br>・敵前衛のいずれか1体を選んで攻撃<br>・特殊能力なし |
| 盾 | 盾を持ち、味方にも盾を付与できる。 | ・バトル開始時から盾1枚を保有する<br>・前衛時：近接攻撃または盾付与のどちらかを選べる<br>・後衛時：盾付与のみ可能 |
| 力 | 他属性より高いBPを持つ | ・前衛にいるときのみ近接攻撃できる（後衛は行動不可）<br>・敵前衛のいずれか1体を選んで攻撃<br>・他属性よりBPが高い<br>・特殊能力なし |
| 弓 | どこからでも弓攻撃できる | ・前衛、後衛どこからでも、好きな敵1体に弓攻撃可能<br>・矢が切れた後は前衛での近接攻撃のみ可能<br>・他属性よりBPが低い |
| 両 | 敵前衛2体に同時に攻撃できる | ・前衛にいるときのみ攻撃できる（後衛は行動不可）<br>・敵前衛の1体に近接攻撃する際、もう片方の敵前衛にも副次攻撃を与える |
| 毒 | 敵に毒の効果を付与できる | ・前衛にいるときのみ近接攻撃できる（後衛は行動不可）<br>・近接でダメージを与えた相手に、毒の効果を付与する |
| 癒 | 味方を治癒・回復できる | ・後衛時：自分か味方を治癒または回復する<br>・前衛時：近接攻撃するか、自分か味方を治癒または回復するかを選べる<br>・治癒・回復はバトル中、2回まで |
| 氷 | 自分に触れた相手を凍結させる | ・前衛にいるときのみ近接攻撃<br>・近接攻撃で自分に触れた相手を凍結させる |
| 嵐 | 嵐を起こして敵2体を攻撃する。 | ・後衛：嵐を起こせる<br>・前衛：近接攻撃か、嵐を起こすかを選べる |
| 忍 | 潜伏状態でバトルを開始する。 | ・潜伏状態でバトルを開始する<br>・前衛にいるときのみ近接攻撃できる |
| 照 | 潜伏状態の敵を照射して暴き出す | ・後衛：潜伏状態の敵忍1体を選んで照射できる<br>・前衛：近接攻撃か照射かを選べる |

- 戦闘エンジンの内部名・フィールド（`stealthActive` 等）は従来どおり。**UI 上の忍の状態名は「潜伏状態」**（§4.10・§5.3）。
- 照属性の行動名は UI 上 **「照射」**（内部 ID は `illuminate`）。

## 3. 属性一覧・解放

### 3.1 一覧

| 内部 ID | 漢字 | 解放 Lv | tie-break 優先度 | 実装 |
|---------|------|---------|------------------|------|
| `attack` | 剣 | 1 | 1 | 済 |
| `defense` | 盾 | 1 | 2 | 済 |
| `power` | 力 | 6 | 3 | 済 |
| `bow` | 弓 | 11 | 4 | 済 |
| `dual` | 両 | 16 | 5 | 済 |
| `poison` | 毒 | 21 | 6 | 済 |
| `heal` | 癒 | 26 | 7 | 済 |
| `ice` | 氷 | 31 | 8 | 済 |
| `storm` | 嵐 | 36 | 9 | 済 |
| `ninja` | 忍 | 41 | 10 | 済 |
| `illuminate` | 照 | 46 | 11 | 済 |

- tie-break: 数値が **大きいほど先** に解決（§7）。
- Lv51 以降: [§11](#11-将来属性草案) の追加属性候補用（[PROTOTYPE §5.9](./PROTOTYPE_DEVELOPMENT_SPEC.md#59-ユーザーレベル解放5nm-統一)）。

### 3.2 作成時の属性

| 項目 | 内容 |
|------|------|
| タイミング | **新規作成時のみ** 自動決定（編集画面では変更不可） |
| 候補 | 作成時点で **解放済み** の全属性（`getUnlockedAttributes(userLevel)`） |
| 抽選 | 各候補の基本权重 **1.0**。**直近解放属性**（スケジュール上、現レベルで最後に追加された属性）のみ **1.1**（他属性比約 +10%） |
| 実装 | `rollAttribute()` / `getAttributeRollWeights()`（`src/card/rollAttribute.ts`）。定数 `ATTRIBUTE_ROLL_RECENT_BOOST = 1.1` |
| 色・hash | **属性決定には使用しない**。BP は従来どおり hash + 密度 + ユーザーレベル（[PROTOTYPE §5.5](./PROTOTYPE_DEVELOPMENT_SPEC.md#55-bp-決定)） |

### 3.3 作成後の属性変更

**実装段階: `v1`（プロトタイプ）** — 経済詳細は [ECONOMY_SPEC §8.4](./ECONOMY_SPEC.md#84-属性変更属性リタッチ属性セレクト)。

| 手段 | コスト | 抽選・選択 | 課金タイミング | BP |
|------|--------|------------|----------------|-----|
| **属性リタッチ** | **300 px** / 回 | §3.2 と同じ权重付きランダム。**同一属性も charged**（モーダルで告知） | リタッチ確定時に **即 px 消費**。デッキ反映はルーレット演出 **完了後** | `recalculateCardBp` |
| **属性セレクト** | **💎 100** / 回 | 解放済み属性から **プレイヤーが1属性指定** | **決定時に即 💎 消費**・即デッキ反映 | 同上（選択中に BP プレビュー表示） |

| 項目 | 内容 |
|------|------|
| 入口 | マイデッキ **カード詳細** — 属性枠右端 **▼** →「属性リタッチ」「属性セレクト」（**ロスト中は非表示**） |
| レア・★・戦績 | **維持** |
| 限界突破 | 属性変更後、専用かけらの対象属性が変わる（[ECONOMY §16.9](./ECONOMY_SPEC.md#169-限界突破ui)） |
| UI レイアウト | 属性枠: 左から **属性アイコン → 説明文 → ▼**（縦中央揃え） |
| 完了 UI | リタッチ: `BP {旧}→{新}` + 保有コイン増減。セレクト: `{ariaName}になりました！` + **BP** `{旧}→{新}(+Δ)` + 保有ジュエル増減（[ECONOMY §16.17](./ECONOMY_SPEC.md#1617-通貨残高の結果表示)） |

- **属性選択チケット**は **採用しない**（属性セレクトと役割が重複。[ECONOMY §10.3](./ECONOMY_SPEC.md#103-属性セレクト)）。
- **カード名変更（リネーム）**は編集画面で **200 px/回**（属性は不変。[ECONOMY §8.2](./ECONOMY_SPEC.md#82-カード名変更編集画面)）。

## 4. 各属性の詳細

### 4.1 剣（attack）— 実装済み

| 位置 | 行動 |
|------|------|
| 前衛 | **近接攻撃**（行動カード）。敵前衛の左右どちらか |
| 後衛 | 行動不可 |

- 特殊能力なし。§2.3 の近接・反撃ルールに従う。
- **UI文案**: §2.5.1（剣）

### 4.2 盾（defense）— 実装済み

| 位置 | 行動 |
|------|------|
| 前衛 | **近接攻撃** または **盾付与** のどちらか1つ（行動カード） |
| 後衛 | **盾付与** のみ（付与可能な対象がいる場合） |

| 項目 | 内容 |
|------|------|
| 初期盾 | バトル開始時から **盾1枚**（前衛・後衛問わず） |
| 盾付与 | 盾なしの **味方1体**（**自分を含む**。自分が盾所持中は対象外）。**1戦闘1回**（防御カードごと） |
| 操作（前衛） | 行動カード選択後、敵前衛タップ＝攻撃、盾なし味方（自分含む）タップ＝盾付与 |

- **UI文案**: §2.5.1（盾）。**盾**・**盾付与**・**近接攻撃** は §2.5.2 の用語モーダル参照。

### 4.3 力（power）— 実装済み

| 位置 | 行動 |
|------|------|
| 前衛 | **近接攻撃**のみ |
| 後衛 | 行動不可 |

| 項目 | 内容 |
|------|------|
| 特殊能力 | **なし**（高 BP で戦う） |
| BP 算出 | 他属性と同様 hash＋密度でレンジ内を決定したうえで、**中心 BP に `POWER_BP_RATIO`（1.5）** を乗じたレンジを使用（[PROTOTYPE §5.5](./PROTOTYPE_DEVELOPMENT_SPEC.md#55-bp-決定)）。レア倍率は他属性と同タイミング |

- **UI文案**: §2.5.1（力）

### 4.4 弓（bow）— 実装済み

| 項目 | 内容 |
|------|------|
| BP 算出 | 剣比 **`BOW_BP_RATIO`（0.52）**。レア倍率は他属性と同タイミング |
| 行動 | **弓攻撃**（前衛・後衛問わず）。矢切れ後は **前衛のみ近接** |
| 弓攻撃 | 生存している **敵1体** を選択。**反撃なし**。与ダメ = currentBp **100%** |
| 弓矢 | **1枚あたり1戦闘2本**。矢切れ後は **弓攻撃不可**（盾で防がれても消費） |
| 対象 | 敵前衛・後衛のいずれも可。**忍のステルス中は狙えない** |
| 近接 | **矢切れ後**、前衛のみ。通常の近接・反撃ルール（§2.3） |
| UI（実装） | 残り矢数の矢バッジ（0 で非表示） |

- **UI文案**: §2.5.1（弓）。**弓攻撃**・**近接攻撃** は §2.5.2 参照。

### 4.5 両（dual）— 実装済み

| 位置 | 行動 |
|------|------|
| 前衛 | **両攻撃**（行動カード：主近接＋副攻撃） |
| 後衛 | 行動不可 |

| 項目 | 内容 |
|------|------|
| 主対象 | 通常の近接（§2.3）。毒・氷の付与も **主対象のみ** |
| 副対象 | **もう片方の敵前衛**（主対象と **同時**） |
| 副ダメージ | 行動開始時の currentBp × **50%**、**無反撃** |
| 撃破との関係 | 主近接の反撃で攻撃側が撃破されても **副攻撃は確定**（同時攻撃のため） |
| 副対象の盾 | あれば **盾消費**（ダメージ 0） |
| 忍 | ステルス中は **近接の主対象にも副対象にもならない** |
| UI（実装） | 主攻撃・副攻撃それぞれに矢印（副攻撃は細線） |

- **UI文案**: §2.5.1（両）。**副次攻撃**・**近接攻撃** は §2.5.2 参照。

### 4.6 毒（poison）— 実装済み

| 位置 | 行動 |
|------|------|
| 前衛 | **近接攻撃** ＋ 毒付与 |
| 後衛 | 行動不可 |

## 5. 状態異常・バフ

### 5.1 毒（スタック）

- `BattleUnit.poisonStacks`: `{ sourceCardId, damagePerTurn }[]`（`src/types/battle.ts`）。
- **付与**: `grantPoisonStack`（`src/game/poisonCombat.ts`）。`damagePerTurn = round(付与時 currentBp × POISON_DOT_RATIO)`。`POISON_DOT_RATIO = 0.3`（`balance.ts`）。
- **DoT**: `startNextTurn` → `applyPoisonDoT`（`src/game/turnPhases/poisonDoT.ts`）。ターン見出しログの直後、行動選択前に適用。
- 毒 DoT で BP 0 になったユニットは即撃破。空き前衛があれば §6 の補充を行動選択前に行う。

### 5.2 凍結

- `BattleUnit.frozenUntilTurn`: 付与時 `state.turn + 2`（`applyFreeze` — `src/game/iceCombat.ts`）。
- **行動不能**: `isFrozen(unit, selectionTurn)` — `selectionTurn = state.turn + 1`（表示ターン）。`selectionTurn <= frozenUntilTurn` の間、行動カード選択不可（`getActionTypesForUnit`）。
- **解除**: `startNextTurn` → `expireFreeze`（`displayTurn > frozenUntilTurn` でクリア）。
- 攻撃対象としては選択可能（§4.8）。盾でダメージ防止時は付与も防止（`meleeAttacks.ts`）。

### 5.3 潜伏状態（忍）

- 内部フィールド: `stealthActive` 等。解除条件は §4.10。
- **UI 表記は「潜伏状態」**（§2.5.2 `stealth`）。コード・ログ上は従来どおり stealth 系名称を使用しうる。
- **癒・盾・嵐・照射・ダメージ全般** で解除。

### 5.4 盾

- 盾属性の初期盾・付与に加え、全属性が `hasShield` を持ちうる。

## 6. ターン処理順

### 6.1 1ターン全体の流れ

**ターン N（N ≥ 2）の開始時** — 行動選択の **前**:

```
① ターン見出し（--- TURN N ---）
② 毒 DoT（全生存ユニット。§5.1）
③ 撃破処理（DoT による撃破）
④ 前衛補充（DoT で空いた前衛。CPU 自動・プレイヤー操作。候補1枚は自動）
⑤ 行動カード選択（各陣営1枚）
   ※ 凍結中のユニットは選択不可（`isFrozen`）
```

**行動確定後**（従来のターン解決）:

```
⑥ 癒（行動カードで癒を選んだ側）
   ※ currentBp 高い順。同率は §7

⑦ 照射（行動カードで照射を選んだ側 — 内部 `illuminate`）
   ※ currentBp 高い順。同率は §7

⑧ 盾付与（行動カードで盾付与を選んだ側）
   ※ currentBp 高い順。同率は §7

⑨ 攻撃フェーズ
   弓・近接・両 等（行動カードの currentBp 高い順。同率は §7）
   ※ 近接は §2.3。複数近接は bidirectional 判定後、1戦闘ずつ
   ※ 嵐は `stormAttacks.ts`（ランダム2体・無反撃）

⑩ 撃破処理（攻撃による撃破）

⑪ 前衛補充（戦闘による空き。CPU 自動・プレイヤー操作）
   ※ 弓が前衛に上がっても BP は変化しない
```

- **ターン 1**: ①は見出しログのみ（既存スタックなし）。②〜④はスキップ相当。
- 実装: ①〜④は `startNextTurn` + UI 再生（`useBattle` の `turnStartPoison`）。⑥〜⑩は `resolveTurn`。
- 前ターン終了後は **⑩⑪**（戦闘起因の撃破・補充）のあと、次ターンの **①〜④** に入る。

### 6.2 補充のタイミング（まとめ）

| きっかけ | タイミング |
|----------|------------|
| 毒 DoT | **行動選択前**（§6.1 ④） |
| 近接・弓・両などの攻撃 | **行動解決後**（§6.1 ⑪） |

## 7. 同率時の解決順（tie-break）

癒・盾付与・攻撃フェーズなど **currentBp 順** を使う箇所で、BP が同率のとき:

| 順位 | 条件 |
|------|------|
| 1 | **currentBp**（高い方が先） |
| 2 | **属性優先度**（§3.1 の表。大きい方が先） |
| 3 | **レア度**（高い方が先: SR > R > N …） |
| 4 | **限界突破 ★**（多い方が先） |
| 5 | **ランダム**（50%。対人戦の公平性のため） |

## 11. 将来属性（草案）

> **ステータス**: アイデア＋設計レビュー。**未実装・数値未確定**。本節の記述は §4 実装済み属性より **優先度が低い**（実装時に改訂する）。

### 11.1 概要

Lv51 以降（[§3.1](#31-一覧)、[PROTOTYPE §5.9](./PROTOTYPE_DEVELOPMENT_SPEC.md#59-ユーザーレベル解放5nm-統一)）向けの **追加属性案**。**照（Lv46）は §4.11 で実装済み**。既存 11 属性と同様 **漢字 1 文字・BP トレードオフ・回数制限・前衛/後衛の役割分担・属性同士のカウンター関係** を目指す。

**全体所感（設計レビュー）**

- 方向性は既存設計と整合。特に **忍（Lv41）への対抗**（解・嵐との三角）と **盤面・属性そのものをいじる系**（替・消・鏡）は、現メタに足りない読み合いの幅を広げうる。

- 経済面: **鏡** は属性セレクト（💎100、[§3.3](#33-作成後の属性変更)）と役割が近い。高 BP 属性の無料コピーを防ぐ弱体化が必要。

### 11.2 属性一覧（案）

| 漢字 | 案の効果（要約） | 既存との関係 | 実装優先度（案） |
|------|------------------|--------------|------------------|
| 蘇 | 撃破後 1 回、死ぬ直前 BP で蘇生 | 護符（戦闘外）・癒と別軸 | 第二候補 |
| 鏡 | 選んだ敵と同じ属性になる | 属性セレクトと被り | 後回し推奨 |
| 爆 | 敵陣全体に currentBp × 20% ダメ | 嵐（ランダム 2 体 ×50%）と住み分け | 第一候補 |
| 重 | 近接で同一対象に 50% ×2 連撃 | 両（別対象 50% 副攻撃）と住み分け | 第二候補 |
| 呪 | 撃破時、キルした相手の BP を 1 に | パッシブ逆転 | 第一候補 |
| 替 | 敵 2 体の配置を入れ替え | 盤面操作（新軸） | 第一候補 |
| 消 | 敵 1 体の属性を消し剣化 | 解・氷と忍/能力対策 | 第二候補 |
| 増 | ターン開始ごとに BP +10% | 毒 DoT と同フェーズ | 第一候補 |
| 鎧 | 被ダメ 2 回まで 50% 軽減 | 盾（1 回完全無効）と住み分け | 第二候補 |
| 先 | BP に関わらず攻撃フェーズで先手 | tie-break（§7）上書き | 第二候補 |
| 解 | 場にいる間、敵忍のステルス無効 | 嵐（能動あぶり出し）と住み分け | 第一候補 |

- **内部 ID・解放 Lv・tie-break 優先度**: 未決定（§10 #7）。
- **実装優先度**: 工数×ゲームへの効きの暫定評価。確定ではない。

### 11.3 各属性の詳細（案）

#### 11.3.1 蘇（revive）— 草案

| 項目 | 案 |
|------|-----|
| 効果 | **1 戦闘 1 回**。撃破されたとき、**撃破直前の currentBp** で蘇生する |
| 位置・行動 | **TBD**（後衛固定蘇生、蘇生直後 1 ターン行動不能など弱体化案あり） |
| BP レシオ（案） | 弓（0.52）より低め。実質 2 体分のため必須枠化を避ける |

**設計レビュー**

- 「死ぬ直前 BP」は直感的。過剰キルされても蘇生後に戦力が残る **リスクテイク型** として面白い。
- 蘇生タイミング（即座 / 次ターン）、前衛/後衛、毒スタック・凍結の引き継ぎ、墓地扱い、蘇生後の「1 戦闘 1 回」カウントを明文化する必要がある。

**TBD**

- 蘇生時のデバフ引き継ぎ
- 毒 DoT・嵐など間接撃破時の蘇生可否
- 蘇生演出と前衛補充タイミング（§6）との整合

#### 11.3.2 鏡（mirror）— 草案

| 項目 | 案 |
|------|-----|
| 効果 | 行動時、**選んだ敵 1 体と同じ属性** になる（能力・BP レシオの扱い **TBD**） |
| 位置・行動 | **TBD**（前衛/後衛、コピー持続ターン数） |
| BP レシオ（案） | 低め。力・忍等のコピーでスイングしすぎないよう制限 |

**設計レビュー**

- アイデアはユニークだが **実装・バランス難度が最大**。全属性の行動・パッシブ・回数カウンタを動的コピーする必要があり、`BattleUnit` の属性別フィールドとの相性が厳しい。
- 属性セレクト（💎）と経済が被る。**v1 簡略案**: 敵の **行動タイプのみ** コピー（BP は鏡自身）とするか、**後回し** を推奨。

**TBD**

- コピー範囲（属性 ID のみ / 行動 / パッシブ / BP 倍率）
- 忍ステルス・力 BP 倍率のコピー時ルール
- 持続（1 ターン / 戦闘終了まで）

#### 11.3.3 爆（bomb）— 草案

| 項目 | 案 |
|------|-----|
| 効果 | 敵陣に爆弾。生存 **敵全員** に currentBp × **20%** ダメージ |
| 位置・行動 | **TBD**（嵐同様、後衛メイン＋前衛 OR 近接 案） |
| 回数（案） | 嵐同様 **1 戦闘 2 回まで** |
| 反撃 | **なし**（§2.3 対象外） |
| 盾（案） | ヒットごとに盾消費（嵐 §4.9 と同様） |
| BP レシオ（案） | 嵐（0.68）より低め（20%×全員は 3〜4 体で実質 60〜80% 相当になりうる） |

**設計レビュー**

- 嵐＝**ランダム 2 体 ×50%**、爆＝**全員 ×20%** で差別化◎。「確実な全体削り」vs「忍あぶり出し＋ランダム」。
- ステルス忍に当たるかで嵐との役割分担が変わる（当たるなら部分重複）。

**TBD**

- ステルス解除の有無（ダメ 0・盾のみでも解除するか）
- 前衛/後衛からの発動可否

#### 11.3.4 重（heavy）— 草案

| 項目 | 案 |
|------|-----|
| 効果 | 前衛・近接時、**同一対象** に currentBp × **50%** を **2 回連続** 攻撃 |
| 位置・行動 | 前衛のみ（両 §4.5 と同型） |
| 反撃 | **TBD**（2 ヒットそれぞれで §2.3 判定するか、1 回分にまとめるか） |

**設計レビュー**

- 両＝**別対象** 50% 副攻撃、重＝**同一対象** 50%×2。トータル 100% なら剣 1 回と同ダメだが、**盾 2 枚消費・反撃 2 回・毒/氷付与 2 回** など副作用が変わる **集中火力** として差別化◎。
- 相打ち（bidirectional）、1 ヒット目で撃破した場合の 2 ヒット目、凍結相手への 2 撃目を定義する必要がある。

**TBD**

- 毒・氷付与は 1 回のみ / 2 回
- BP レシオ（剣並〜やや低）

#### 11.3.5 呪（curse）— 草案

| 項目 | 案 |
|------|-----|
| 効果 | **パッシブ**。このカードを **撃破した相手** の currentBp を **1** にする |
| 位置・行動 | 通常近接（剣同型）＋撃破時トリガー |
| BP レシオ（案） | 低め（0.5 前後）。捨て駒/罠属性 |

**設計レビュー**

- 高 BP の力・増を「交換で潰す」カウンターとして個性的◎。
- **「撃破した相手」** の定義が重要。推奨: このカードが **最後の 1 ダメージ** を与えた相手に限定（毒 DoT・嵐・両副攻撃等との整理）。

**TBD**

- 同時ダメージ（両副攻撃等）でのキル判定
- 呪発動後も相手が生存している場合（BP 1 で残る）の UI

#### 11.3.6 替（swap）— 草案

| 項目 | 案 |
|------|-----|
| 効果 | **敵 2 体** の **配置（前衛左右 / 前衛↔後衛）** を入れ替える |

| 回数（案） | **1 戦闘 1〜2 回** |

**設計レビュー**

#### 11.3.7 消（erase）— 草案

| 項目 | 案 |

**設計レビュー**

- 氷（1 ターン行動不能）・癒（デバフ解除）とは別の **永続弱体化** として強力。
- 忍対策として **解（オーラ）・嵐（能動）・消（単体永久）** の 3 層になる。役割分担を明文化推奨:
  - **解** = パッシブ全体（ステルス無効）
  - **消** = 単体永久（全能力剥奪）
  - **嵐** = 能動＋ダメ＋あぶり出し

| 初回無反撃 | **TBD** — 推奨: **ステルス無効のみ**。初回無反撃は封じない（忍の半分の柱を残す） |

**設計レビュー**

- 忍（Lv41）への **デッキ構築型カウンター** として非常に良い ◎。**能動解除は照（§4.11）**、**パッシブ解除は本属性（解）** で住み分け可能。嵐はランダム能動あぶり出し（§4.9）。

### 11.4 重複整理と段階投入（案）

#### 11.4.1 既存属性との住み分け

| 提案 | 既存 | 住み分け |
|------|------|----------|
| 爆 | 嵐 | 爆＝確実・低倍率・全員 / 嵐＝ランダム・高倍率・あぶり出し |
| 重 | 両 | 重＝単体集中 / 両＝前衛拡散 |
| 鎧 | 盾 | 鎧＝2 回半減 / 盾＝1 回完全無効 |
| 解・消 | 嵐 | 解＝パッシブ全体 / 消＝単体永久 / 嵐＝能動＋ダメ |
| 鏡 | 属性セレクト | 経済と被る。弱体化 or 見送り |

#### 11.4.2 段階投入バンドル（推奨案）

| バンドル | 属性 | 意図 |
|----------|------|------|
| **A — コントロール** | 替・消・解 | 盤面・能力・忍対策 |
| **B — 成長・パッシブ** | 増・呪・蘇 | 時間軸・逆転・生存 |
| **C — 攻撃バリエーション** | 重・爆・鎧・先 | 既存攻撃ループの拡張 |

- 鏡は **バンドル外（後回し）** を推奨。
- 各バンドル 2〜3 属性を Lv46 以降の 5n+m スケジュールに **1 バンドルずつ** 載せる案。

#### 11.4.3 実装優先度（暫定）

**第一候補**（既存フレームに載せやすくメタ拡張効果が大きい）: 解 → 替 → 呪 → 増 → 爆

**第二候補**（ルール定義を詰めてから）: 重 → 消 → 鎧 → 蘇 → 先

**後回し推奨**: 鏡（全属性コピーは工数・バグリスク最大）
