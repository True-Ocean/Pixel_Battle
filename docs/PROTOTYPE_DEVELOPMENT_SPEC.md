## 1. 目的とスコープ

### 1.1 目的

- 企画「簡単！真剣！お絵描きピクセルバトル！」の**コア体験**を、低コストなウェブプロトタイプで検証する。
- **5枚制限・前衛2/後衛3の配置・ターン戦闘・自作ドット絵・カード名**の感触を確認する。
- 戦闘ロジックを **UI と分離** し、将来の **Unity 移植** および **サーバー検証** に再利用しやすくする。

### 1.2 プロトタイプで作るもの

| 区分 | 内容 |
|------|------|
| プラットフォーム | ブラウザ（React + Vite + TypeScript） |
| データ保存 | `localStorage`（サーバーなし） |
| 対戦 | CPU のみ |
| レベル | Lv0 固定（攻撃・防御、色：白・黒・赤、キャンバス 16×16） |

### 1.3 プロトタイプで作らないもの

- ★（スター）・ユーザーレベルアップ
- 遠隔先制・復活・電撃・氷・両刀・毒 など Lv1 以降の属性
- 遠距離攻撃（弓など）・後衛への直接攻撃
- 本番ロスト（カードデータの永久削除）
- 課金・殿堂入り・オンライン非同期対戦
- 画像ファイルのサーバー保存

## 2. 技術スタックと環境

### 2.1 採用技術

| 層 | 技術 |
|----|------|
| フロントエンド | React 18+ / Vite / TypeScript |
| 状態・UI | React（コンポーネント + state） |
| 永続化 | `localStorage` |
| パッケージ管理 | npm |

### 2.2 開発環境

- **Node.js**: インストール済み想定（確認時 v24.14.1）
- **エディタ**: Cursor / VS Code
- **初回セットアップ（実装時）**:
  ```bash
  npm create vite@latest . -- --template react-ts
  npm install
  npm run dev
  ```
- **スマホ実機確認**: 開発サーバーは `host: true` で LAN 上の端末からアクセス可能（例: `http://192.168.x.x:5173`）
- **PWA**: `vite-plugin-pwa` によりホーム画面追加に対応。アイコン生成は `npm run generate:pwa-icons`

### 2.3 推奨ディレクトリ構成

```
src/
  components/     # 画面（キャンバス、デッキ、戦闘UI、ConfirmDialog）
  config/         # balance.ts / palette.ts（定数・パレット配置）
  game/           # 純粋な戦闘ロジック（UI非依存）
  canvas/         # 16×16 描画・flood fill・チェッカーボード
  card/           # 属性・HP 算出（色バランス + hash）、create / update
  storage/        # localStorage 読み書き
  types/          # Card, Attribute, BattleState 等
```

**原則**: `game/` と `card/` は React に依存しない関数・型のみとし、Unity（C#）移植時は仕様書 + テストケースを参照して再実装する。

## 3. 将来アーキテクチャ（参考）

本プロトタイプの決定事項は、以下の将来構成と矛盾しないこと。

| 段階 | 技術 | 役割 |
|------|------|------|
| プロトタイプ | React + Vite | ルール検証・バランス・UX |
| 本番クライアント | Unity | バトルアニメ・パーティクル・ドット演出 |

## 4. 機能一覧（プロトタイプ）

### 4.1 ユーザーフロー

1. **マイデッキ**画面（最大5枚表示・並べ替え・削除・タップで編集）
2. **イメージ作成** / **カード編集**：16×16 キャンバス、パレット（白・黒・赤 + ツール）、**カード名入力** → 生成または保存
3. 戦闘開始：5枚を **前衛2枚・後衛3枚** に配置 → バトル開始
4. ターン制戦闘（CPU 対戦）
5. 勝敗 → 仮ロスト演出（該当時）→ 次戦闘前にデッキ自動復帰

### 4.2 画面要件（最低限）

- **マイデッキ**一覧（5枚上限、新規作成・編集・削除・並べ替え）
- ドット描画エディタ（パレット・消しゴム・塗りつぶし・クリア）+ 名前入力 + 生成（属性・HP は自動算出）
- 戦闘前：5枚配置 UI（前衛2・後衛3）+ 30秒準備タイマー
- 5枚通しの盤面、ターンごとの行動カード選択、対象選択、**段階的な攻撃演出**（ダメージ表示 → HPカウントダウン → 次の攻撃）
- 中央ガイダンス帯（1行表示）、陣営別勝敗表示（WIN / LOSE）、終了後フッター操作
- プロトタイプ注記：「カードは保存されています（仮ロスト）」

### 4.3 マイデッキ画面 UI

| 項目 | 内容 |
|------|------|
| 画面タイトル | **マイデッキ** |
| サブ情報 | `最大5枚 · N / 5`。仮ロスト中は「仮ロスト演出中（データは保存済み）」を表示 |
| 通常時の操作 | カード行をタップ → **カード編集**画面へ。サブに「タップで編集」を表示 |
| 編集モード | ヘッダ右の **編集** / **完了** トグル。サブに「並べ替え・削除」を表示 |
| 並べ替え | ドラッグ＆ドロップ、または各行の **↑** / **↓** |
| 削除 | 行左の **−** → 確認ダイアログ（「戦績も失われます」）。確定で `localStorage` から削除 |
| 編集中の制限 | カードタップ編集・**新規カード作成**・**バトル** を無効化 |
| フッター操作 | **新規カード作成**（5枚未満）、**バトル**（5枚揃ったときのみ有効） |
| 5枚未満 | 「戦闘にはあと N 枚必要です（合計 5 枚）」を表示 |

### 4.4 イメージ作成 / カード編集画面 UI

| 項目 | 新規（イメージ作成） | 編集（カード編集） |
|------|----------------------|---------------------|
| 画面タイトル | **イメージ作成** | **カード編集** |
| 確定ボタン | **カード作成**（確認ダイアログ後に保存） | **保存**（即時） |
| 戻る | **マイデッキに戻る**（未保存内容は破棄） | 同左 |

**レイアウト（上から）**

1. 画面タイトル（この画面ではアプリ共通ナビ **非表示**）
2. **パレット** + **16×16 キャンバス**（縦方向に余白を確保）
3. **カード名** 入力欄 + 説明文
4. フッター固定：**カード作成** / **保存**、**マイデッキに戻る**

**説明文（カード名の下）**

> あなたが描いたイメージとカード名から、カードのステータスが自動生成されるよ

**パレット（2×8 グリッド）**

| 行 | 内容 |
|----|------|
| 1行目 | 利用可能な色（Lv0 は **白・黒・赤** の3色のみ。未解放色のロック枠は表示しない） |
| 2行目 | **消しゴム** \| **塗りつぶし** \| **クリア** |

| ツール | 挙動 |
|--------|------|
| 色スウォッチ | 選択色で **ペイント** モードに切替 |
| 消しゴム | タップしたマスを `null`（透明）に |
| 塗りつぶし | 同色相連領域を **flood fill** |
| クリア | 全マスを空に（チェッカーボード背景 + 「クリア」ラベル） |

**キャンバス**

- 16×16。未塗りマスはセル単位の **チェッカーボード** 表示
- ペイント / 消しゴムはドラッグで連続操作（塗りつぶしは1タップ）

**バリデーション（確定前）**

| 条件 | メッセージ |
|------|------------|
| カード名が空 | カード名を入力してください |
| 塗りマス 0 | 1マス以上塗ってください |
| デッキ上限（新規のみ） | デッキは最大 5 枚です |

**新規作成の確認ダイアログ**

- タイトル: 「カードを作成しますか？」
- 本文: 「このイメージと名前でカードを作成します。よろしいですか？」
- 確定: **作成する**（primary スタイル） / キャンセル

**編集時のデータ保持**

- `updateCardFromDrawing`：`id`・`wins`・`losses`・`createdAt` は維持
- `name`・`pixels` に応じて `attribute`・`hp` を再算出

### 4.5 戦闘準備・戦闘画面 UI

| 項目 | 内容 |
|------|------|
| 画面上部タイトル | **「バトル準備」「バトル！」は表示しない**（アプリ共通ヘッダも非表示） |
| 中央ガイダンス | 盤面中央の **1行帯**（ピンク〜紫のグラデーション）。ターン・操作ヒントを `TURN N：…` 形式で1行表示 |
| 準備時の文言例 | `バトル準備：カードを配置してください（残り N 秒）` |
| バトル時の文言例 | `TURN 1：カードを選択` / `TURN 3：攻撃先か盾先を選択` / `TURN 8：バトル終了！` |
| スロット枠 | 配置中は空スロットに点線枠＋「前衛」「後衛」ラベル。双方5枚配置完了後は **カードありスロットの枠を非表示**。バトル中も同様 |
| 敵手札・配置 | 上部に敵手札5枚（裏面）を表示。CPU は手札からスロットへ **1枚ずつ順番に配置アニメ** |
| 配置完了ボタン | 画面下部に固定（`ランダム配置` / `準備完了`）。盤面・手札は必要に応じてスクロール |
| 防御・前衛の操作 | **行動ボタンは置かない**。行動カード選択後、**敵前衛タップ＝攻撃**、**盾なし味方タップ＝盾付与** |
| 試合終了 | 中央は `TURN N：バトル終了！` のみ。**WIN / LOSE は各陣営エリア中央に大きく重ね表示**（上＝敵、下＝味方）。対人戦では各クライアントの上下を「自分／相手」に割り当てる |
| 終了後操作 | `デッキ` / `新規バトル` を画面下端に固定。盤面はスクロール可能 |

## 5. カード作成仕様

### 5.1 入力

| 項目 | 必須 | 説明 |
|------|------|------|
| `name` | ○ | プレイヤーが付けるカード名 |
| `pixels` | ○ | 16×16、色は白・黒・赤、未塗りは `null`（または透明指定） |

### 5.2 色比率の算出

塗られたマスのみを対象とする（未塗りは分母に含めない。実装時にコメントで固定すること）。

```
R = 赤マス数 / 塗ったマス合計
W = 白マス数 / 塗ったマス合計
B = 黒マス数 / 塗ったマス合計
（R + W + B = 1）
```

### 5.3 シード（hash）

- **入力文字列**: `name` + 正規化した `pixels`（JSON 文字列など、実装で1形式に固定）
- **処理**:  cryptographic hash または十分な衝突耐性の hash → 数値化（例: 0〜1 の `hashAttack`, `hashDefense`）
- **性質**: 同じ名前 + 同じ絵 → **常に同じ** 属性・HP（決定論）

### 5.4 属性決定（Lv0: 攻撃 | 防御）

**方針**: 最多色ではなく **3色バランスと hash を常に合成**（Q14-B）。

```
色攻撃 = R + B × BLACK_ATTACK_WEIGHT   // 初期 0.7（黒は攻撃寄り）
色防御 = W + B × (1 - BLACK_ATTACK_WEIGHT)  // 初期 0.3

最終攻撃 = 色攻撃 × COLOR_WEIGHT + hashAttack × HASH_WEIGHT
最終防御 = 色防御 × COLOR_WEIGHT + hashDefense × HASH_WEIGHT

// 初期: COLOR_WEIGHT = 0.7, HASH_WEIGHT = 0.3（定数1か所、後から調整可）

attribute = 最終攻撃 >= 最終防御 ? "attack" : "defense"
```

### 5.5 HP 決定

- **方針**: `hash` + **密度**（塗ったマス / 256）で、属性ごとの **HP レンジ** 内の値を決定。
- 密度が高いほどレンジの **上限側** に寄せる（線形補間等、実装時に `src/config/balance.ts` に定義）。
- プロトタイプ用の具体レンジ（例、要プレイテスト）:

| 属性 | HP レンジ（案） |
|------|-----------------|
| attack | 70 〜 100 |
| defense | 55 〜 85 |

※上記数値は指示書時点の仮値。`src/config/balance.ts` で一元管理すること。

### 5.6 Lv0 パレット

| 色 | 用途 |
|----|------|
| 白 | 色比率・防御寄りスコア |
| 黒 | 色比率・攻撃寄り（重み 0.7） |
| 赤 | 色比率・攻撃寄り |

**UI 上のパレット配置（Lv0）**

- グリッドは **2 行 × 8 列**（`PALETTE_GRID_ROWS` × `PALETTE_GRID_COLS`）
- Lv0 では先頭 **3 色のみ** 表示（`PALETTE_UNLOCKED_COUNT_LV0 = 3`）。将来の16色拡張用スロットは Lv1 以降で解放
- 2 行目左から **消しゴム・塗りつぶし・クリア**（描画ツール。色選択とは独立）
- 将来 Lv アップ時は 1 行目に色を追加し、2 行目ツール位置は固定（`paletteToolPlacement`）

## 6. デッキと永続化

### 6.1 デッキルール

- 保有上限: **5枚**
- ストックなし（常に5枚デッキのみ）

### 6.2 保存形式（Q18-C）

**永続化する（`localStorage`）**

```typescript
interface Card {
  id: string;              // uuid 等
  name: string;
  pixels: (string | null)[][];  // 16×16、例: "#FF0000" | null
  attribute: "attack" | "defense";
  hp: number;              // 戦闘時の最大HPとして使用
  wins: number;
  losses: number;
  reviveCount: number;     // プロトタイプでは 0 固定で可
  createdAt: string;       // ISO 8601
}

interface SaveData {
  deck: Card[];            // 最大5件
}
```

**永続化しない（戦闘セッション内メモリのみ）**

- 現在 HP
- 盾の付与状態・使用済みフラグ
- 麻痺・毒等（将来）

### 6.3 ピクセル保存の段階

| 段階 | 形式 |
|------|------|
| プロトタイプ | JSON 2次元配列 |
| 本番 / DB | RLE 等へ変換（同じ論理データ） |

### 6.4 デッキ操作（マイデッキ）

| 操作 | 仕様 |
|------|------|
| 追加 | **新規カード作成** → イメージ作成画面。上限 5 枚 |
| 編集 | カード行タップ → カード編集画面。保存で上書き |
| 削除 | 編集モードの **−** → 確認後に削除（`wins` / `losses` も失う） |
| 並べ替え | 編集モードでドラッグまたは ↑↓。順序は `localStorage` に保存 |
| 表示順 | 並べ替え後の配列順がマイデッキ一覧・戦闘前手札の初期順に反映 |

## 7. 戦闘前フロー

1. プレイヤーの **5枚** と CPU の **5枚** を表示する。
2. スマホ縦持ちを最優先し、上部に敵カード列（裏面5枚）、下部に自分カード列（表面5枚）を小さく横並び表示する。
3. 中央盤面に、敵・自分それぞれ **前衛2スロット / 後衛3スロット** を表示する。
4. CPU は準備開始時に5枚の配置をランダム決定する。上部の敵手札（裏面5枚）から、空の敵スロットへ **前衛左 → 前衛右 → 後衛左 → 後衛中央 → 後衛右** の順で1枚ずつ配置アニメーションする。
5. プレイヤーは下部カード列からタップ方式で自分の5枚を中央の自分スロットへ配置する。
6. 準備時間は **30秒**。準備画面表示と同時にカウント開始する。
7. 5スロット未配置の間、ボタンは **「ランダム配置」** と表示する。押すと未配置カードのみを空きスロットへランダム配置し、すぐには開始しない。
8. 5スロットすべて配置済みになると、ボタンは **「準備完了」** に変わる。押すとバトル開始。
9. 30秒経過時は、未配置カードを空きスロットへランダム配置し、そのままバトル開始する。
10. バトル開始時、CPUカードを **前衛左 → 前衛右 → 後衛左 → 後衛中央 → 後衛右** の順に1枚ずつ表面へ反転する。ガイダンスは **「カードオープン」** とする。

### 7.1 配置と操作

| 項目 | ルール |
|------|--------|
| 初期配置 | 前衛2枚、後衛3枚 |
| 自分の盤面向き | 前衛2枚が中央寄り、後衛3枚が手前 |
| 敵の盤面向き | 前衛2枚が中央寄り、後衛3枚が奥 |
| 操作方式 | ドラッグではなくタップ方式 |
| 選択解除 | 選択中の同じカードを再タップ |
| 未配置カード → 空きスロット | 配置 |
| 未配置カード → 埋まっているスロット | 入れ替え（元カードは未配置列へ戻る） |
| 配置済みカード → 空きスロット | 移動 |
| 配置済みカード → 埋まっているスロット | 入れ替え |
| 戻し操作 | 配置済みカードを未配置列へ戻す専用操作はなし |
| 時間切れランダム配置 | 既に配置済みのカード位置は動かさない |
| 手札表示 | 未配置カードは属性ラベル付きで表示。配置のたびに手札は中央揃え |
| カウントダウン | 中央ガイダンス帯に1行で表示（`バトル準備：…（残り N 秒）`） |
| スロット枠 | 配置中は点線枠を表示。双方5枚配置完了後、カードありスロットの枠は非表示 |
| 画面タイトル | 「バトル準備」「バトル！」の画面上部タイトルは表示しない |

### 7.2 用語

| 用語 | 意味 |
|------|------|
| 行動カード | そのターンに自分が選んで行動させる1枚 |
| 前衛 | 近接攻撃できる2スロット |
| 後衛 | 通常の近接攻撃はできない3スロット |
| 墓地 | 撃破されたカードを小さく並べる表示領域 |
| 盾 | 1回だけ受けるダメージを0にして消費される状態 |

## 8. 戦闘ルール

### 8.1 試合の流れ

- **1ターン** を繰り返す。
- **終了条件**: どちらか一方の **5枚すべてが HP 0**。
- **勝利**: 相手の5枚すべて撃破。
- **敗北**: 自分の5枚すべて撃破。
- 撃破されたカードは盤面スロットから外れ、墓地へ移動する。元スロットは空きスロットになる。

### 8.2 ターン開始：行動カード選択

| ルール | 内容 |
|--------|------|
| 選択枚数 | 1ターンに行動できるカードは各陣営 **1枚だけ** |
| プレイヤー | 行動可能な自分カードから1枚を選ぶ |
| CPU | 行動可能なCPUカードから1枚を選ぶ |
| 同時戦闘 | プレイヤーとCPUの行動は同時選択・同時解決として扱う |
| 攻撃属性・前衛 | 攻撃のみ選択可能。敵前衛の左右どちらかを対象に選ぶ |
| 攻撃属性・後衛 | 近接攻撃しかできないため選択不可 |
| 防御属性・前衛 | 攻撃 または 盾付与 のどちらかを選ぶ（行動ボタンは使わない） |
| 防御属性・後衛 | 盾付与が可能な場合のみ選択可能。攻撃はできない |
| 防御・前衛の操作 | 行動カード選択後、**敵前衛タップ＝攻撃**、**盾なし味方タップ＝盾付与**。ガイダンスは `攻撃先か盾先を選択` |

### 8.3 攻撃対象

| 攻撃種別 | 対象 |
|----------|------|
| 近接攻撃 | 敵前衛の生存カード。自分の左右位置に関係なく、敵前衛の左右どちらも攻撃可能 |
| 遠距離攻撃 | 将来拡張。弓属性などで後衛から敵を攻撃可能にする想定 |

現時点では近接攻撃のみ。後衛カードが直接攻撃で倒されるケースは発生しないが、将来の遠距離攻撃に備えて後衛死亡も状態として扱える設計にする。

### 8.4 ターン処理：フェーズ

```
フェーズ① 行動選択
  - プレイヤーが行動カード・行動種別・対象を選ぶ
  - CPU も行動候補から選ぶ

フェーズ② 攻撃前能力
  - 防御: 味方1体へ盾を付与
  - 同じターンに付与された盾も、そのターンの攻撃を防げる

フェーズ③ 攻撃
  - 攻撃行動がある場合、**相互ダメージ** を解決する
  - 攻撃側は相手の現在HP分のダメージを与える
  - 防御側（被攻撃側）も攻撃側の現在HP分のダメージを受ける
  - 盾がある側はそのダメージを0にし、盾を消費する
  - 攻撃側自身が盾を持っている場合、反撃ダメージは0になり、攻撃側の盾が消費される
  - 盾同士の戦いでは両方の盾が消え、双方HPは変わらない
  - 複数の攻撃が同時に発生する場合、**1戦闘ずつ順番に処理** する
  - 1枚が複数の相手と戦う場合、**相手HPが低い順** に処理する
  - 処理順は UI 演出（ダメージ表示・HP変化）にも反映する

フェーズ④ 撃破・墓地移動
  - HP 0 のカードを墓地へ移動し、元スロットを空きにする

フェーズ⑤ 前衛補充
  - 空いた前衛スロットへ、対応する後衛カードを上げる
  - 補充が完了するまで次ターンに進まない
```

### 8.5 前衛補充

配置を以下のように表す。

```
前衛: A B
後衛: C D E
```

| 空き前衛 | 上がれる後衛 |
|----------|--------------|
| A（左前衛） | C（左後衛）または D（中央後衛） |
| B（右前衛） | D（中央後衛）または E（右後衛） |

- D は左右どちらにも上がれる共有候補。
- 片側だけ空いている場合、対応候補がいなければ空きのまま次ターンへ進む。
- 両前衛が空いた場合も同じ対応ルールを守る。
- 複数候補がある場合、プレイヤーは補充順と移動先を自由に選べる。
- 候補が **1枚だけ** の場合は自動で前衛へ補充する（ユーザー操作不要）。
- 候補が **2枚** の場合のみ、ユーザーに選択を促す。
- CPU は対応候補からランダムに上げる。
- 補充操作はタップ方式: 上げる後衛カードをタップ → 同じカードを再タップで確定（1回目で選択、2回目で確定）。
- ガイダンス文言:
  - 未選択時: **「前衛に移動する後衛カードを選択」**
  - 選択後: **「もう一度タップで確定」**

### 8.6 撃破後

- HP 0 のカードはその試合では **もう選択できない**。
- 撃破カードは墓地に小さく表示し、グレーアウトまたは撃破ビジュアルを付ける。
- 自分と敵の墓地は分けて表示する。
- 盤面レイアウトは、準備中・バトル中・墓地移動後も固定する。

### 8.7 試合終了表示

| 項目 | 内容 |
|------|------|
| 中央ガイダンス | `TURN N：バトル終了！`（勝敗文言は出さない） |
| 敵陣営エリア | 墓地＋敵スロット全体の中央に **WIN** または **LOSE** を大きく重ね表示 |
| 味方陣営エリア | 味方スロット＋墓地全体の中央に **WIN** または **LOSE** を大きく重ね表示 |
| スタイル | グローは使わず、鮮やかな塗り＋太い縁取り（WIN＝黄、LOSE＝赤系） |
| 対人戦（将来） | 各クライアントで自分の陣営側に WIN/LOSE を表示。中央は双方共通の終了メッセージ |
| 終了後操作 | `デッキ` / `新規バトル` を画面下端に固定。盤面本体はスクロール可能 |

### 8.8 攻撃解決の詳細

#### 相互ダメージ

近接攻撃が成立した場合、攻撃側・防御側は **互いの現在HP分** のダメージを受ける。

例: カードA（HP 80）がカードX（HP 85）を攻撃する場合

| カード | 受けるダメージ | 結果HP |
|--------|----------------|--------|
| A | 85（Xの現在HP） | 0 |
| X | 80（Aの現在HP） | 5 |

#### 盾の相互作用

| 状況 | 結果 |
|------|------|
| 防御側のみ盾あり | 防御側はダメージ0、盾消費。攻撃側は通常通り反撃ダメージ |
| 攻撃側のみ盾あり | 攻撃側は反撃ダメージ0、盾消費。防御側は通常通りダメージ |
| 双方とも盾あり | 双方の盾が消え、双方HPは変わらない |

#### 複数攻撃の処理順

同一ターンに複数の近接攻撃が発生する場合:

1. 互いに同じ2枚のカードが攻撃し合う場合は **1戦闘** としてまとめる（両矢印）。
2. 残りの戦闘は **1件ずつ順番** に処理する。
3. **1枚が複数の相手と戦う場合**、そのカードから見て **相手HPが低い順** に処理する。
4. 上記以外の戦闘同士の順序は、双方の最大HPが高い順 → 同率なら防御側HPが高い順。

UI 演出もこの処理順に従い、各戦闘ごとに「ダメージ表示 → HPカウントダウン」を行う。

## 9. 属性効果（Lv0）

### 9.1 攻撃

- 前衛にいる場合のみ選択可能。
- 行動は近接攻撃のみ。
- 敵前衛の生存カードから左右どちらかを対象に選ぶ。
- 後衛にいる場合は、攻撃対象が目の前にいないため選択不可。

### 9.2 防御

| 項目 | ルール |
|------|--------|
| 初期盾 | 防御属性カードは、前衛・後衛に関係なくバトル開始時から盾を1つ持つ |
| 行動選択 | 1ターン中に **攻撃** または **盾付与** のどちらか一方だけを選ぶ |
| 前衛 | 攻撃または盾付与を選択可能 |
| 後衛 | 盾付与が可能な場合のみ選択可能。攻撃はできない |
| 盾付与対象 | 盾を持っていない生存中の味方1体。**自分自身は不可** |
| 対象不可 | 既に盾を持っている味方は選べない |
| 盾の効果 | **1回限り**、ダメージを0にする（付与された盾は1回被弾で消える） |
| 攻撃時の盾 | 盾を持った状態で攻撃すると、**攻撃側の盾は必ず消費** される（反撃ダメージは0） |
| 盾同士 | 双方の盾が消え、双方HPは減らない |
| 付与能力 | **防御カードごとに、1戦闘で1回だけ** 盾付与可能 |
| 再付与 | 同じ防御カードが別ターンで再度付与することは不可 |
| 攻撃との関係 | 盾付与を選んだターンは攻撃しない。攻撃を選んだターンは盾付与しない |
| 両陣営が盾付与 | 攻撃なしで、両方の盾付与だけ処理してターン終了 |

### 9.3 属性優先度表

プロトタイプでは最小限。拡張時に表形式で管理する。

| 優先度（小さいほど先） | 属性 | フェーズ |
|------------------------|------|----------|
| 10 | defense（盾付与） | 攻撃前能力 |
| 20 | ranged（遠距離攻撃） | 攻撃 ※将来 |
| 30 | attack / defense（近接攻撃） | 攻撃 |

## 10. CPU 仕様

### 10.1 配置

- CPU の5枚配置は、プロトタイプではランダム。
- 準備中は上部の敵手札に裏面5枚を表示し、空スロットへ1枚ずつ飛ばして配置する。
- 配置順は **前衛左 → 前衛右 → 後衛左 → 後衛中央 → 後衛右**。
- バトル開始時に同順で1枚ずつ表面へ反転する。

### 10.2 行動選択

- 初期実装では、CPUの通常ターン行動はランダムでよい。
- 実行可能な行動候補を列挙し、その中からランダムに1つ選ぶ。
- 攻撃行動の場合、敵前衛の生存カードからランダムに対象を選ぶ。
- 盾付与行動の場合、盾を持っていない生存味方からランダムに対象を選ぶ（**付与者自身は除く**）。
- 将来は、前衛保護・低HP保護・撃破可能攻撃などを評価するスコア式AIへ差し替えてよい。

### 10.3 前衛補充

- 空いた前衛に対して、対応候補からランダムに上げる。
- 候補がいなければ補充しない。

### 10.4 CPU デッキ

- プロトタイプ: 生成ルールまたは `src/data/cpu-decks.json` 等の **固定デッキ** で可。
- 将来: 他ユーザーの5枚メタデータを取得し、同じルールで CPU が操作。

### 12.6 画面・演出指針

#### レイアウト

- スマホ縦持ちを最優先レイアウトとする。
- PC では body 背景の上に **phone シェル** として中央配置する（`--phone-shell-w: 26.5rem`、バトル画面は `--phone-shell-battle-w: 22rem`）。
- 盤面レイアウトは準備中・バトル中・墓地移動後も固定する。
- 手札・配置スロット・バトル中・墓地の **すべてのカード** は、同一の内部レイアウト（パーツ位置・サイズ・マージン）で表示する。
- カードサイズは `--formation-card-w` / `--formation-card-h` で統一する。
- 中央はガイダンス表示と矢印・ダメージ演出のために使う。

#### イメージ作成 / カード編集画面

- 画面上部は共通ブランドタイトルのみ。**画面切替ナビは非表示**。
- パレット → キャンバス → カード名 → 説明文の間に **約 0.7〜0.85rem** の縦余白を確保する。
- フッター（**カード作成** / **保存**、**マイデッキに戻る**）は画面下部に固定配置する。

#### 戦闘準備・戦闘画面

- 戦闘準備・戦闘中は画面上部の「バトル準備」「バトル！」タイトルおよびアプリ共通ヘッダを **非表示** とする。
- 中央ガイダンスは **1行帯**（ピンク〜紫グラデーション、左右アクセントライン）。`TURN N：操作ヒント` 形式で常に1行。
- 準備フェーズは盤面・手札を `formation-setup-body` でスクロール可能にし、下部の `ランダム配置` / `準備完了` は固定。
- バトル終了後は盤面を `formation-battle-body` でスクロール可能にし、`デッキ` / `新規バトル` を下端固定。
- スロット枠は配置中のみ点線表示。双方配置完了後およびバトル中は、カードありスロットの枠を非表示（`is-frameless`）。
- 試合終了時、中央は `バトル終了！`、各陣営エリア（`formation-field-cpu` / `formation-field-player`）に WIN/LOSE を重ねる。

#### ガイダンス・操作

- ガイダンス文言は短い日本語で統一する。
- ターン表示は `TURN 1` のような英字表記とする。
- 行動カード選択中は選択カードをハイライト表示する。
- 選択中の同じカードを再タップすると選択解除する（専用の「選択解除」ボタンは置かない）。
- 防御属性・前衛で攻撃と盾付与の両方が可能なとき、**行動ボタンは出さない**。敵前衛タップで攻撃、盾なし味方タップで盾付与。
- 攻撃対象選択時は **敵前衛** を赤系でハイライトする。盾付与対象は **味方** を緑系でハイライトする。

#### 盾・バフ表示

- 盾状態はカード全体を覆う半透明表示ではなく、**カード左上の小さなバフアイコン**（🛡）で示す。
- 将来のバフ・デバフも同じ左上エリアに並べられる構造とする。
- HP は右上にライトな色のバッジ表示とする。

#### ターン演出（攻撃・盾）

1. **盾付与フェーズ**（該当時）  
   緑系の矢印で付与元→付与先を表示する。

2. **攻撃フェーズ**（1戦闘ずつ順番に）  
   各戦闘について次の順で再生する。

   | サブフェーズ | 内容 |
   |--------------|------|
   | ダメージ表示 | 関係するカードの中央に `−85` などのダメージ数値を大きく表示。盾で0の場合は非表示 |
   | HP変化 | ダメージ表示を消し、右上HPを **from→to** でカウントダウンアニメーション |
   | 次へ | 上記完了後、次の戦闘へ。全戦闘完了後にターン終了処理へ |

3. **矢印**
   - 攻撃は **カードの端から端** へ向ける（中央同士ではなく、矩形外周の交点を始点・終点とする）。
   - 相互攻撃の場合は両矢印（↔）表示。
   - 対象が異なる同時攻撃は片矢印2本を同時表示。
   - **HPカウントダウン中は攻撃矢印を非表示** にする（ダメージ表示フェーズでのみ表示）。
   - ダメージ表示は矢印レイヤーより上の専用レイヤーに描画し、矢印に隠れないようにする。

4. **その他**
   - 処理中のカードは揺れエフェクトを付ける。
   - バトルログは画面に出さない。内部ログ・構造化イベントはリプレイ用に維持する。

#### タイミング定数（案）

| 定数名 | 初期値 (ms) | 説明 |
|--------|-------------|------|
| `CLASH_MS.damage` | 420 | ダメージ数値表示時間 |
| `CLASH_MS.hp` | 520 | HPカウントダウン時間 |
| `CLASH_MS.exit` | 450 | ターン演出終了後の待機 |

#### resolveTurn 戻り値（攻撃演出用）

各 `AttackPlayback` には以下を含める。

- 攻撃側・防御側の位置（`fromSide` / `fromPosition` / `toSide` / `toPosition`）
- 防御側ダメージ・HP変化（`damage`, `hpFrom`, `hpTo`）
- 攻撃側ダメージ・HP変化（`attackerDamage`, `attackerHpFrom`, `attackerHpTo`）
- 相互攻撃フラグ（`bidirectional`）
- 処理後状態（`stateAfter`）
